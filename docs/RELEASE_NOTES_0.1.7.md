# Velorn v0.1.7 Draft Release Notes

## Choose The Right Download

- **Desktop app:** for most users who want to install Velorn and use the editor, Generate workflows, the embedded ComfyUI tab, and export tools directly.
- **Workflow Starter Pack (optional):** for advanced ComfyUI users who want to inspect setup workflows manually in ComfyUI and prepare nodes/models outside the app.

## Highlights

- New built-in ComfyUI launcher controls, setup guidance, and partner API key UX
- New auto-import bridge that pulls successful generations from the embedded ComfyUI tab back into the asset panel
- Better long-running workflow reliability and clearer ComfyUI generation logs during runs
- Polished welcome screen, safer context menus, and improved local video defaults with `LTX 2.3` first

## ComfyUI Launcher And Workflow Setup

- Velorn now has a richer in-app ComfyUI launcher experience, including clearer startup state, better log visibility, and tighter wiring between the Electron shell and the renderer
- The app now includes a more guided workflow setup experience for local workflows, with setup gallery improvements, install catalog support, and clearer dependency surfacing
- Partner/API-key-oriented UX has been improved so cloud/partner-node workflows are easier to understand and configure from inside the app
- The Generate and settings flows now do a better job explaining what runs locally versus what uses partner credits

## Embedded ComfyUI Auto-Import

- Successful prompts run in the embedded ComfyUI tab can now be auto-imported back into Velorn's asset panel
- Imported media is placed into a dedicated `Imported from ComfyUI` area so it stays separate from assets generated by Velorn-managed workflows
- Imported assets get workflow sidecar metadata for better traceability
- Image sequences can be auto-detected and stitched into video assets when they look like frame-based output rather than unrelated still-image batches
- Stitched sequence imports now show a badge and can be unstitched back into individual frames from the asset panel

## Reliability And Generation Feedback

- Long-running local workflows, especially heavy image-to-video runs like `WAN 2.2`, no longer time out as aggressively while ComfyUI is still actively working
- The app is better at distinguishing a genuinely failed ComfyUI execution from a slow workflow that is still rendering
- The launcher log viewer now surfaces more useful generation events so users can see what ComfyUI is doing during a run
- Additional launcher integration now forwards relevant workflow/runtime events into the in-app log tail instead of leaving them hidden in the background process

## Local Video Workflow Improvements

- `LTX 2.3` is now presented ahead of `WAN 2.2` in the main image-to-video workflow picker
- Director Mode beta local video defaults now favor `LTX 2.3`, which is much faster on lighter hardware while still keeping `WAN 2.2` available as an explicit choice
- Director Mode FPS handling now correctly applies to both local `LTX 2.3` and local `WAN 2.2` renders

## UI And Editing Polish

- The welcome screen project-selection experience has been refreshed with a stronger hero treatment, cleaner project panel presentation, and improved readability
- Recent project cards now support better thumbnail handling
- Right-click menus in the timeline and preview are more viewport-aware, which helps prevent menus and submenus from falling off-screen or getting clipped
- Mask-related preview behavior is more stable, including fixes for black masked regions and more reliable layered compositing behavior

## Captions And Media Handling

- Caption/transcription workflow plumbing has been improved further, including timeline-audio mixing support used by the ComfyUI transcription path
- Caption rendering/export support continues to evolve with related renderer and asset-handling improvements in this release

## Workflow Starter Pack

- The Workflow Starter Pack remains optional for advanced users who want to inspect or prepare workflows manually outside the app
- This `v0.1.7` release is primarily focused on the desktop-app experience; if the starter-pack artifact is refreshed separately, attach the updated zip that matches this release

## Important Setup Note

Velorn generation still depends on a separate local ComfyUI installation in this build.

- Local workflows may require manual node/model setup.
- Cloud workflows still use local ComfyUI and may require partner nodes plus a Comfy account API key.
- The Workflow Starter Pack remains optional and is mainly for advanced users who want to inspect or prepare workflows manually.

## Known Limitations

- This is still a pre-release style workflow-heavy desktop app.
- ComfyUI connections are local-only in this build.
- Some workflows still require manual node/model setup in ComfyUI.
- Cloud pricing and partner workflow requirements may vary by provider.
- GPU encoding depends on an NVENC-capable FFmpeg build and supported NVIDIA hardware.
